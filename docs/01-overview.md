# 01 — Product Overview

## Vision Statement

> **Guild is a self-hosted multi-agent development OS for solo developers.**
> You describe the goal. Your agents — persistent, private, and autonomous — figure out the execution.

Guild transforms how solo developers work by replacing the cycle of context-switching, micromanaging AI sessions, and losing progress between sessions. Instead of prompting a single AI assistant repeatedly, Guild gives you a permanent team of specialized agents that remember your codebase, collaborate on tasks, and operate autonomously while you focus on higher-level thinking.

---

## Problem Statement

Solo developers managing multiple projects face a compounding set of challenges that existing AI tools do not address:

- Every new Claude Code session starts from zero — no memory of past decisions, no codebase knowledge, no context
- Running multiple AI sessions simultaneously requires constant manual supervision and coordination
- There is no audit trail of what the AI has decided, changed, or learned
- Context loss between sessions means the developer must re-explain the same things repeatedly
- Existing tools are single-agent — they scale the individual, not the workflow

> **Core insight:** The bottleneck is not AI capability. It is coordination, memory, and continuity. Guild solves all three.

---

## Positioning

| Guild **is** | Guild **is not** |
|---|---|
| A self-hosted multi-agent orchestration system | A cloud AI service |
| A persistent memory layer for your codebase | A code editor or IDE plugin |
| An autonomous development team on your machine | A replacement for developer judgment |
| A git-native workflow manager | A wrapper around another AI tool |
| A proactive chief of staff for your projects | A team collaboration platform |

---

## Target User

Guild v1.0 is designed exclusively for the **solo developer** profile:

- Manages 2–6 concurrent projects across different clients or domains
- Comfortable with terminal, git, and developer tooling
- Values privacy — unwilling to send codebase to third-party cloud services
- Already uses or is open to using Claude Code as primary AI coding tool
- Spends significant time context-switching rather than executing

> Explicitly out of scope for v1.0: teams, non-technical users, enterprise, mobile.
